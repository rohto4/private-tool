# 2026-04-28 作業記録 草案

## 1. 概要

Private-Tool PJで、RSS/ニュースFeedを取得してSlackの用途別チャンネルへ自動投稿するSlack Botを作った。
Botは `#feed-ai`、`#feed-tech`、`#feed-other` へ記事を振り分け、Slack投稿内にMisskey.io共有ボタンとMisskey API投稿ボタンを持つ。
DBはNeon PostgreSQL + Prismaの軽量構成にし、Feed取得元、取得記事、配送履歴、ジョブ履歴を保存する形にした。
GitHub Actionsから毎時取得・投稿し、毎日21時ごろに取得数/投稿数/スキップ数のサマリを流す前提まで整えた。
ローカル初回実行では6 sourceから125件取得し、121件をDB保存し、Slackへ20件投稿できた。

## 3. 作ったもの

- Feedを取得して定期的にSlackチャンネルへ自動投稿するSlack Botを作った。
  - 取得記事をキーワードで `#feed-ai`、`#feed-tech`、`#feed-other` に振り分ける。
  - Slack投稿にMisskey.io共有画面を開くボタンと、Misskey APIでワンクリック投稿するボタンを付ける。
  - Neon PostgreSQLにFeed取得元、取得記事、Slack/Misskey配送履歴、ジョブ実行履歴を保存する。
  - GitHub Actionsで毎時実行でき、手動では `/pt-fetch` slash commandから起動できる仕様にした。
  - 毎日21時ごろ、当日の取得数、投稿数、スキップ数を `#feed-other` に投稿する仕様にした。
- 実装前提、実行手順、不足部品、進捗をdocs配下に整理した。

## 10. 注意点

- Neon接続URLで `sslmode=require` を使うと、pg/pg-connection-stringの将来変更に関する警告が出る。
- 可能なら `sslmode=verify-full` に変更する。
- Slack slash command/interactivityはdeploy先の公開URL設定が必要。
- GitHub Actionsで動かすにはrepository secrets設定が必要。

## 11. 日誌ルール整備

- セッション終了前、または大きな作業区切りで [日誌](./20260428.md) を残す運用を定めた。
- 日誌ルールは [PJ文脈](../../PROJECT.md) に記録した。
- 日誌ファイルは `docs/diary/YYYYMMDD.md` 形式にする。
- 日誌には概要、目次、作ったもの、主要変更、実行コマンド、検証結果、注意点、残タスクを含める。
- 概要は文書全体の量に応じて書く。今回程度なら5行程度を目安にする。
- 目次は `###` 階層まで含め、見出しには項番を付ける。
- ファイル参照は `[役割](相対パス)` 形式にし、役割ラベルは10文字以内を目安にする。
- コマンド、チャンネル名、環境変数名など、ファイルではないものはバッククォート表記でよい。
- [Docs索引](../README.md) に `diary/` の用途も追加した。
