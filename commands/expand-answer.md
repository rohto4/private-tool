# expand-answer

直前または指定された回答を、ユーザが判断しやすい形に展開する。

## 目的

通常の一次回答では省略した背景、判断理由、比較、リスク、次の選択肢を補足する。
常に長くするための command ではなく、必要な部分だけを詳しくする。

## 方針

- まず結論を短く書く。
- 対象領域はユーザの質問内容と現在の PJ 文脈に従う。
- PJ 固有の領域は、関係する場合だけ扱う。
- 事実、推論、提案を分ける。
- 既に合意済みの前提や明らかな一般論は繰り返さない。
- 比較する場合は、判断に効く軸だけ出す。
- 長くなる場合は、最初に要点を置く。
- 実装・設定・運用に関わる場合は、採用した場合と採用しない場合の差分を書く。
- 最後に、次に決めるべきことを 1〜3 個に絞る。

## 避けること

- 関係しない PJ 固有情報を持ち出す。
- 一般論を長く書く。
- 同じ説明を繰り返す。
- 詳細化の名目で、論点を勝手に広げる。
- まだ採用していない仕組みを前提にする。

## 出力

内容に応じて、次から必要な項目だけ使う。

1. 結論
2. 補足背景
3. 判断理由
4. 比較
5. リスク
6. 推奨
7. 次に決めること

## 使い方

ユーザが `/expand-answer`、`expand-answer`、または「さっきの回答を詳しく」のように依頼した時に使う。

展開対象が曖昧な場合は、原則として直前の assistant 回答を対象にする。
複数候補があり誤解のリスクが高い場合だけ、どの回答を展開するか確認する。
